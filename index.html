1 The Big Idea ~5 min

As it actually is (exam version)

Let S be a closed surface enclosing a volume V. If F̄ is a continuously differentiable vector point function, then:

The total flux of F̄ leaving through the closed surface S equals the total divergence (outflow-per-point) added up over the entire volume V inside.

In short: what leaks out through the skin = what's being generated inside.

Like you're 5

Imagine a closed room full of kids blowing up balloons and letting them go — puff! — into the air.

You could chase down every single puff of air inside the room and add them up. Or, you could just stand at the door and windows and count all the air breezing past you on its way out.

Both counts give the exact same number. Puffs made inside = air that escapes outside.

2 The Formula ~5 min

∫∫∫V div F̄ dV  =  ∫∫S F̄ · n̂ dS (also written ∭V ∇·F̄ dV = ∬S F̄·dS̄)

n̂ = the outward-drawn unit normal at any point of S. This is the detail students lose marks on — it must point away from the solid, always.

Cartesian form (what you'll actually compute with)

Let F̄ = f₁ î + f₂ ĵ + f₃ k̂ and n̂ = cos α î + cos β ĵ + cos γ k̂ (direction cosines of n̂). Then:

∭V ( ∂f₁/∂x + ∂f₂/∂y + ∂f₃/∂z ) dx dy dz  =  ∬S f₁ dy dz + f₂ dz dx + f₃ dx dy

The left side is called div F̄ or ∇·F̄. It's just three partial derivatives added together — that's it.

Like you're 5

div F̄ is just: "how hard is the balloon puffing right at THIS exact spot?" You work that out for every tiny spot in the room, then add them all up — that's the left side.

The right side is the grown-up way of saying "count the air escaping through every window and door." Same total, promise.

n̂ is just which way you're facing when you stand at a window — always facing outward, away from the room, never inward. That way you never count the same puff of air twice.

3 Why It's True — Picture It ~5 min

The trick behind the proof

Chop volume V into tiny cubes. Each tiny cube leaks a bit of flux out of its 6 faces.

When two tiny cubes sit side by side, the flux leaving one face exactly cancels the flux entering its neighbor's matching face (equal and opposite) — so all the internal faces cancel out.

Only the outer faces — the ones on the boundary S — survive. So: sum of tiny outflows (divergence, everywhere inside) = flux through only the outside skin.

Like you're 5

Now imagine the balloon-room is actually built out of lots of tiny toy blocks stacked together, like Lego.

Each little block passes its puff of air to the block right next to it. Whatever one block gives away, its neighbor receives — so those two cancel out and don't matter.

The only puffs that don't get cancelled are the ones at the very outside edge of the whole pile, because there's no neighbor block out there to catch them. So all that's left, in the end, is exactly what escapes through the big outside wall.

4 The Method You'll Actually Use ~5 min

Every "verify the theorem" question follows the same recipe:

  1. RHS side (easy side): compute div F̄ = ∂f₁/∂x + ∂f₂/∂y + ∂f₃/∂z, then triple-integrate it over the solid's limits.
  2. LHS side (longer side): break the closed surface into its flat/curved faces. For each face: find n̂ (outward), compute F̄·n̂, substitute the face's fixed coordinate, then integrate over that face's 2D region.
  3. Add up all the faces — this total must equal the RHS number.
  4. State: "∴ LHS = RHS ⇒ Gauss's divergence theorem is verified."

sphere/cylinder x²+y²+z²=a² (or similar φ=0)n̂ = ∇φ / |∇φ|

Like you're 5

Think about wrapping a present in a box. To know how much wrapping paper you used, you don't do it all at once — you check one flat side at a time (top, bottom, front, back, left, right), then add up all six pieces of paper.

That's the entire method. The "easy side" (RHS) is like weighing the whole present at once. The "six faces" (LHS) is wrapping it side by side. Both better give the same answer — that's what you're proving.

5 Full Worked Example — the Cube ~7 min

Verify Gauss's theorem for F̄ = (x³ − yz) î − 2x²y ĵ + z k̂ over the cube 0 ≤ x,y,z ≤ a. (Click each row to expand.)

Like you're 5 — the same example, no scary letters

Picture a toy cube-shaped room, like a big box, with sides of length "a". Inside, there's a magic wind machine — it doesn't blow the same everywhere, it blows harder in some corners and softer in others.

We want to check one thing: does "all the wind puffed up inside" match "all the wind leaking out through the 6 walls"?

Inside check: we add up how hard the wind is puffing at every single tiny spot in the room. We get one number.

Outside check: we go wall by wall — right wall, left wall, front wall, back wall, ceiling, floor — and measure how much wind leaks through each one. Two of the six walls (the left one and the back one) happen to leak exactly nothing, which saves us work. We add up all six.

Both checks land on the exact same number: a⁵/3 + a³. That's the whole trick — no matter how fancy the wind pattern looks with all those x's and y's, inside-count always equals outside-count.

Now the full working, step by step:

Step 1 · RHS — div F̄
div F̄ = ∂/∂x(x³−yz) + ∂/∂y(−2x²y) + ∂/∂z(z) = 3x² − 2x² + 1 = x² + 1
Step 2 · RHS — triple integral
∭(x²+1) dx dy dz over 0→a each = a·a·[x³/3 + x]₀ᵃ = a² · (a³/3 + a) = a⁵/3 + a³
Step 3 · LHS — six faces
Face x=a: ∬(a³−yz)dydz = a⁵ − a⁴/4
Face x=0: ∬yz dydz = a⁴/4
Face y=a: ∬−2x²a dxdz = −2a⁵/3
Face y=0: 0
Face z=a: ∬a dxdy = a³
Face z=0: 0
Step 4 · Add the six faces
(a⁵ − a⁴/4) + a⁴/4 + (−2a⁵/3) + 0 + a³ + 0 = a⁵ − 2a⁵/3 + a³ = a⁵/3 + a³
Step 5 · Conclusion
LHS = a⁵/3 + a³ = RHS ⇒ Gauss's divergence theorem is verified. This exact cube setup is the most common exam question — memorize the flow, not just the numbers.

6 Quick Second Pattern — the Sphere ~3 min

Compute ∬(ax² + by² + cz²) dS over the sphere x²+y²+z² = 1.

Trick: write it as F̄·n̂ where F̄ = ax î + by ĵ + cz k̂ (since n̂ = x î + y ĵ + z k̂ on the unit sphere).

div F̄ = a + b + c → constant, so the integral is just (a+b+c) × Volume of sphere = (a+b+c) × 4π/3.

Answer: 4π(a+b+c)/3 — no messy surface splitting needed when n̂ ∝ position vector, like a sphere centered at the origin.

Like you're 5

Now swap the toy cube room for a big round beach ball. This time the wind always blows straight outward, like the ball itself is slowly inflating — every spot pushes directly away from the center.

Because the wind and the wall are always facing the exact same way, you don't need to check patches one at a time like the six walls of the cube. You just take "how hard it's puffing on average" (a+b+c) and multiply by "how big the ball is inside" (4π/3). One multiplication, done.

Green's Theorem

The 2D cousin of Gauss/Stokes — swaps a walk around a flat region's edge for what's swirling inside it.

The Big Idea

As it actually is (exam version)

If S is a closed region in the xy-plane bounded by a simple closed curve C, and M, N are continuous functions of x, y with continuous partial derivatives in R, then:

Green's theorem turns a line integral around the boundary C into a double integral over the flat region S it encloses.

Like you're 5

Picture a flat garden with a fence around it. Green's theorem says: instead of walking all the way around the fence checking the wind pushing you forward or back, you can just add up all the little "swirls" of wind happening everywhere inside the garden.

Walking the fence and counting swirls inside both give the exact same number.

The Formula

∮C (M dx + N dy)  =  ∬S ( ∂N/∂x − ∂M/∂y ) dx dy

C must be traversed counter-clockwise (the positive direction) for this sign convention. M and N are just the x-part and y-part of what you're integrating along the curve.

Like you're 5

∮ means "walk all the way around the fence and add up the push you feel at every step." ∬ means "stand inside the garden and add up every tiny swirl happening there instead."

Walking counter-clockwise just means keeping the garden on your left the whole way round — that's the rule that makes both sides match.

Full Worked Example

Verify Green's theorem for ∮(3x² − 8y²)dx + (4y − 6xy)dy, where C is the closed curve bounding the region between y = √x and y = x².

Like you're 5

Imagine a curvy leaf-shaped garden bed, pinched between two curved fence lines that cross at (0,0) and (1,1). "Swirl-counting" inside the leaf should match "fence-walking" around its edge — that's all we're checking.

Step 1 · RHS — find ∂N/∂x − ∂M/∂y
M = 3x²−8y², N = 4y−6xy. ∂M/∂y = −16y, ∂N/∂x = −6y. So ∂N/∂x − ∂M/∂y = −6y −(−16y) = 10y.
Step 2 · RHS — double integral over the leaf region
Curves meet at (0,0) and (1,1), region is x²≤y≤√x, 0≤x≤1.
∫₀¹∫x²√x 10y dy dx = 5∫₀¹(x − x⁴)dx = 5[x²/2 − x⁵/5]₀¹ = 5(3/10) = 3/2.
Step 3 · LHS — line integral along OA (y = x²)
dy = 2x dx. ∫₀¹(3x²−8x⁴)dx + (4x²−6x³)(2x)dx = ∫₀¹(3x²+8x³−20x⁴)dx = [x³+2x⁴−4x⁵]₀¹ = −1.
Step 4 · LHS — line integral along AO (x = y²)
dx = 2y dy, y goes 1→0. Working out ∫(3y⁴−8y²)(2y)dy + (4y−6y³)dy = 5/2.
Step 5 · Add and conclude
LHS = −1 + 5/2 = 3/2 = RHS ⇒ Green's theorem is verified.

Stokes' Theorem

The 3D upgrade of Green's — swaps a walk around a curved surface's rim for the spin happening across it.

The Big Idea

As it actually is (exam version)

Let S be an open surface bounded by a closed, non-intersecting curve C. If F̄ is a differentiable vector point function, then:

C is traversed in the positive direction and n̂ is the unit outward normal at any point of S. Setting the surface flat onto the xy-plane (n̂ = k̂) actually reduces Stokes' theorem to Green's theorem — they're the same idea, one dimension apart.

Like you're 5

Picture a butterfly net — a piece of cloth stretched tight over a bent wire hoop. Walk your finger all the way around the wire and feel how much it twists. Or, drop lots of tiny toy pinwheels all over the cloth and add up how much each one spins.

Both ways give you the exact same number.

The Formula

∮C F̄ · dr̄  =  ∬S (curl F̄) · n̂ dS

curl F̄ = ∇×F̄ measures the "spin" of the field at each point of S; you're adding all those tiny spins up and comparing to the total twist felt walking the rim C.

Like you're 5

∮F̄·dr̄ means "walk around the wire hoop and feel the twist." ∬(curl F̄)·n̂ dS means "drop pinwheels all over the cloth and add up every little spin."

curl F̄ is just a spin-checker: point it at one exact spot, and it tells you if a pinwheel there would spin, and which way.

Full Worked Example

Verify Stokes' theorem for F̄ = −y³ î + x³ ĵ, where S is the circular disc x²+y² ≤ 1, z = 0.

Like you're 5

Here the butterfly net cloth is stretched flat, like a flat circle. Walking around its rim should feel like the exact same amount of twist as adding up every pinwheel's spin across the flat cloth.

Step 1 · LHS — set up the rim walk
C: x=cosθ, y=sinθ, 0→2π. F·dr = −y³dx + x³dy = sin⁴θ dθ + cos⁴θ dθ (after substitution).
Step 2 · LHS — evaluate the integral
∫₀2π(sin⁴θ+cos⁴θ)dθ = ∫₀2π(1 − 2sin²θcos²θ)dθ = 2π − ½∫₀2πsin²2θ dθ = 2π − π/2 = 3π/2.
Step 3 · RHS — find curl F̄
F = −y³î + x³ĵ + 0k̂. curl F̄ = ∇×F̄ = (3x²+3y²)k̂ = 3(x²+y²) k̂.
Step 4 · RHS — integrate over the disc (polar coordinates)
n̂ = k̂, so ∬3(x²+y²) dS = 3∫₀2π∫₀¹ r²·r dr dθ = 3·2π·[r⁴/4]₀¹ = 3π/2.
Step 5 · Conclude
LHS = RHS = 3π/2 ⇒ Stokes' theorem is verified.

Gradient, Divergence & Curl

The three basic operations ∇ can do — the toolkit every theorem above is built from.

The Big Idea — three tools, one operator ∇

∇ = î ∂/∂x + ĵ ∂/∂y + k̂ ∂/∂z  (the "del" operator)

Gradient — ∇φ (on a scalar)

grad φ = ∇φ = î ∂φ/∂x + ĵ ∂φ/∂y + k̂ ∂φ/∂z. Output is a vector. Directional derivative of φ at a point, along unit vector ê, is ∇φ·ê. The greatest possible rate of increase at a point is |∇φ|.

Like you're 5

If φ is the height of a hill, ∇φ is an arrow at your feet that always points straight uphill — and how long the arrow is tells you how steep that hill is right there.

Divergence — ∇·F̄ (on a vector)

div F̄ = ∇·F̄ = ∂f₁/∂x + ∂f₂/∂y + ∂f₃/∂z. Output is a scalar. F̄ is called solenoidal if div F̄ = 0 everywhere.

Like you're 5

Stand at one spot in a windy field. Divergence tells you if air is being born right there (a tiny spring, positive) or disappearing (a tiny drain, negative).

Curl — ∇×F̄ (on a vector)

curl F̄ = ∇×F̄, computed as a determinant. Output is a vector. F̄ is called irrotational if curl F̄ = 0 — and then F̄ = ∇φ for some scalar potential φ.

Like you're 5

Drop a tiny toy pinwheel at that exact spot. Curl just tells you whether it spins, and how fast.

Three Mini Worked Examples

Like you're 5 — gradient example

You're standing on a bumpy hill at one exact spot, facing one exact direction. "Directional derivative" just answers: if I take one step that way, do I go up or down, and how steeply? That's it — the scary formula is just a shortcut for that one question.

Gradient · directional derivative
φ = x²−y²+2z² at P(1,2,3), direction 4î−2ĵ+k̂.
∇φ = 2xî−2yĵ+4zk̂ → at P: 2î−4ĵ+12k̂.
ê = (4î−2ĵ+k̂)/√21.
Directional derivative = ∇φ·ê = (8+8+12)/√21 = 28/√21.

Like you're 5 — divergence example

You're standing at one exact spot (1,−1,1) in a windy field made by this formula. Divergence just asks: right here, is wind being born (a mini fountain) or sucked away (a mini drain)? A positive answer (9) means it's a little fountain at that spot.

Divergence · find div f̄ at a point
f̄ = xy²î + 2x²yzĵ − 3yz²k̂, find div f̄ at (1,−1,1).
div f̄ = y² + 2x²z − 6yz.
At (1,−1,1): 1 + 2(1)(1) − 6(−1)(1) = 1+2+6 = 9.

Like you're 5 — curl example

We drop a tiny pinwheel everywhere in this field and check: does it ever spin? We do the math and get 0 everywhere — meaning it never spins, anywhere. When a field never spins, it's secretly hiding a simple "hill height" map underneath it (the scalar potential φ) — like the whole windy field was really just "which way is uphill" in disguise.

Curl · show F̄ is irrotational, find its scalar potential
F̄ = (x²−yz)î + (y²−zx)ĵ + (z²−xy)k̂.
curl F̄ = i(−x+x) − j(−y+y) − k(−z+z) = 0 ⇒ F̄ is irrotational.
Since F̄ = ∇φ, integrating each component gives φ = x³/3 + y³/3 + z³/3 − xyz + C.

Surface Integrals

The technique hiding inside every LHS you computed in Gauss's and Stokes' examples.

The Big Idea

As it actually is (exam version)

A surface integral ∬S F̄·n̂ dS adds up F̄·n̂ (the part of F̄ poking through the surface) across every point of a curved or flat surface S.

Since S is usually curved, you can't integrate over it directly — you project it onto a flat coordinate plane and integrate there instead.

Like you're 5

Hold your hand up in the sunshine and look at its shadow on the ground. Your hand is bumpy and curved, but its shadow is flat and easy to trace. That's the whole trick — measure the easy flat shadow instead of the tricky curved thing.

The Formula

∬S F̄ · n̂ dS  =  ∬R ( F̄ · n̂ ) / |n̂ · k̂|  dx dy

R is the "shadow" (projection) of S onto the xy-plane. If n̂·k̂ = 0 anywhere on S (the surface is edge-on to that plane), project onto a different coordinate plane instead — use whichever plane the surface doesn't stand perpendicular to.

Like you're 5

R is just the shadow on the ground. If you tilt your hand sideways, its shadow gets smaller and skinnier than your real hand — dividing by |n̂·k̂| stretches that skinny shadow back out to the true size before measuring.

If you turn your hand so its edge faces the sun, the shadow disappears completely (that's n̂·k̂ = 0) — so you'd trace the shadow on a side wall instead.

Full Worked Example

Evaluate ∬S 3(x²+y²) k̂ · n̂ dS, where S is the flat disc x²+y² ≤ 1 in the plane z = 0. (This is exactly the RHS piece from the Stokes' worked example.)

Like you're 5

Here the hand is already flat and lying right on the ground — so its shadow is just its own exact shape, no stretching needed.

Step 1 · Set n̂ and simplify
Surface is flat in the xy-plane, so n̂ = k̂ and n̂·k̂ = 1 (no correction factor needed). The integral becomes ∬R 3(x²+y²) dx dy over the unit disc R.
Step 2 · Switch to polar coordinates
x²+y² = r², dx dy = r dr dθ, r: 0→1, θ: 0→2π.
Step 3 · Integrate
3∫₀2π∫₀¹ r²·r dr dθ = 3·2π·[r⁴/4]₀¹ = 3·2π·(1/4) = 3π/2.

Volume Integrals

Adding a scalar or vector function up across a solid region — pick the coordinate system that matches its shape.

The Big Idea

As it actually is (exam version)

A volume integral ∭V f dV sums f throughout every tiny piece of a 3D solid V. The trick is choosing dV to match the solid's shape.

Like you're 5

It's like counting how many little blocks fill up a shape. A box: stack them in straight rows. A can: stack them in circles. A ball: stack them in layers, like peeling an orange from the outside in. Same total blocks, just an easier way to count for each shape.

The Formula (pick your coordinates)

ShapeCoordinate systemdV
Box / cubeCartesian (x,y,z)dx dy dz
CylinderCylindrical (r,θ,z)r dr dθ dz
Ball / sphereSpherical (r,θ,φ)r² sinθ dr dθ dφ

Like you're 5

Cartesian = scooping sand in straight little cubes, like Lego bricks — good for boxes.

Cylindrical = scooping in ring-shaped slices, like cutting a can into coins then wedges — good for anything round around one straight axis.

Spherical = counting in layers around a center point, like peeling an orange — good for balls. The extra "r² sinθ" just makes up for the outer peels being bigger than the inner ones, so the count comes out right.

Full Worked Example

Evaluate ∭V 4z dV, where V is the upper half of the ball x²+y²+z² ≤ a² (the solid sitting on the xy-plane, capped by the upper hemisphere). This is the exact volume integral that comes out of applying Gauss's theorem to F̄ = yz²î + zx²ĵ + 2z²k̂.

Like you're 5

Picture half an orange sitting flat-side-down, like a little bowl. We count it in peeled layers from the center out (spherical coordinates) since that matches the round shape perfectly, and we only go halfway up (θ stops at 90°) since it's half an orange, not a whole one.

Step 1 · Switch to spherical coordinates
x = r sinθ cosφ, y = r sinθ sinφ, z = r cosθ, dV = r² sinθ dr dθ dφ. Since it's the upper half-ball only: r: 0→a, θ: 0→π/2 (not π — that would be the full ball), φ: 0→2π.
Step 2 · Substitute z = r cosθ
∭4z dV = 4∫₀a∫₀π/2∫₀2π (r cosθ)(r² sinθ) dφ dθ dr = 4∫₀ar³dr · ∫₀π/2sinθcosθ dθ · ∫₀2πdφ.
Step 3 · Evaluate each piece
∫₀ar³dr = a⁴/4.   ∫₀π/2sinθcosθ dθ = [sin²θ/2]₀π/2 = 1/2.   ∫₀2πdφ = 2π.
Step 4 · Multiply it all together
4 · (a⁴/4) · (1/2) · 2π = π a⁴.

Laplace Convolution Theorem

Turns a messy "mixing" of two functions into simple multiplication.

The Big Idea

As it actually is (exam version)

The convolution of f(t) and g(t) is (f*g)(t) = ∫₀ᵗ f(τ) g(t−τ) dτ — a way of "blending" two functions together, common in systems/signals problems.

The Convolution Theorem says: taking the Laplace transform of that blend is the same as just multiplying their individual transforms.

Like you're 5

Imagine two cups of juice, orange and apple. Slowly pouring them together and figuring out the exact taste at every moment is complicated. But if you just look at the number written on each cup's label and multiply those two numbers, you get the same answer — much easier.

The Formula

L{ (f*g)(t) }  =  F(s) · G(s) where F(s) = L{f(t)}, G(s) = L{g(t)}, and (f*g)(t) = ∫₀ᵗ f(τ)g(t−τ)dτ

Read backwards, this is the most useful direction on exams: L⁻¹{F(s)G(s)} = (f*g)(t) — it lets you invert a product of two known transforms without partial fractions.

Like you're 5

Slowly pouring two juices together and tasting the blend = looking at each juice's label number and multiplying them. Same answer either way.

The useful trick: if someone hands you two label numbers already multiplied together, you can always work backwards to find the two juices that made them — no messy extra steps.

Full Worked Example

Find L⁻¹{ 1 / [(s−a)(s−b)] } using the convolution theorem (a ≠ b).

Like you're 5

1/(s−a) is the label number for juice eat, and 1/(s−b) is the label number for juice ebt. Since we're given the two label numbers already multiplied together, we just work backwards to find how those two juices blend.

Step 1 · Identify f(t) and g(t)
F(s) = 1/(s−a) → f(t) = eat.   G(s) = 1/(s−b) → g(t) = ebt.
Step 2 · Set up the convolution integral
(f*g)(t) = ∫₀ᵗ eaτ eb(t−τ) dτ = ebt ∫₀ᵗ e(a−b)τ dτ.
Step 3 · Integrate
ebt · [ e(a−b)τ/(a−b) ]₀ᵗ = ebt · [e(a−b)t − 1]/(a−b) = [eat − ebt]/(a−b).
Step 4 · Conclusion
L⁻¹{ 1/[(s−a)(s−b)] } = (eat − ebt) / (a−b).

Pointers & Pointer Arithmetic (C)

A variable that stores an address instead of a value — and the arithmetic rules that come with it.

The Big Idea

As it actually is (exam version)

A pointer is a variable that stores the memory address of another variable, instead of storing a value directly. It's declared with a *: int *p; means "p is a pointer to an int."

& means "address of." * (in front of an existing pointer) means "the value stored at this address" — called dereferencing.

Like you're 5

Imagine houses on a street, each with its own house number. A regular variable is the person living in the house. A pointer is just a sticky note with a house number written on it — not the person.

If you walk to that house number and knock (dereference), you find the actual person.

The Formula (syntax + pointer arithmetic)

int x = 10;
int *p = &x;      // p now holds the address of x
printf("%d", *p);  // dereference: prints the value at that address -> 10
p = p + 1;         // jumps forward by sizeof(int) bytes, not 1 byte
SymbolMeaning
&xaddress of variable x
*pvalue stored at the address p holds
p + 1address of the next element of that type (not +1 byte)
p - 1address of the previous element of that type

Like you're 5

& is asking "what's your house number?" * is asking "who lives at this house number?"

Adding 1 to a pointer doesn't move one step down the street — it jumps to the next whole house of the same size. If houses (ints) are each 4 steps wide, p+1 jumps 4 steps, not 1.

Full Worked Example (compiled & run)

A program that walks through an array using pointer arithmetic, then swaps two numbers using a function that takes pointers.

Like you're 5

ptr is a sticky note holding the address of the very first box in a row of toy boxes (the array). *(ptr+i) means: walk i boxes down the row from that starting address, and peek inside.

For the swap: instead of handing the function copies of your two numbers (which it can't actually change), you hand it the two house addresses — now it can walk over and really swap what's inside.

Step 1 · The code
#include <stdio.h>

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main() {
    int arr[5] = {10, 20, 30, 40, 50};
    int *ptr = arr;

    for (int i = 0; i < 5; i++) {
        printf("*(ptr+%d) = %d\n", i, *(ptr + i));
    }

    int x = 5, y = 10;
    printf("Before swap: x = %d, y = %d\n", x, y);
    swap(&x, &y);
    printf("After swap: x = %d, y = %d\n", x, y);
    return 0;
}
Step 2 · What each line does
int *ptr = arr; — arrays decay to a pointer to their first element, so ptr already points at arr[0].
*(ptr + i) — steps i elements forward from ptr and reads the value there. Identical to arr[i].
swap(&x, &y) — passes the addresses of x and y, not copies. Inside swap, *a and *b reach back and change the originals.
Step 3 · Actual program output
*(ptr+0) = 10 *(ptr+1) = 20 *(ptr+2) = 30 *(ptr+3) = 40 *(ptr+4) = 50 Before swap: x = 5, y = 10 After swap: x = 10, y = 5
This was actually compiled and run to confirm the output — it's exact, not guessed.

Basic String Functions (C)

Strings are just character arrays with a secret stop sign at the end — here's the toolkit for handling them.

The Big Idea

As it actually is (exam version)

In C, a string is simply an array of characters ending with a special null character '\0' that marks where the string stops. There is no separate "string type."

The <string.h> library gives you ready-made functions so you don't have to write character-by-character loops yourself every time.

Like you're 5

A string is like a train of letter-cars lined up in a row. The very last car is a secret "stop" car (\0) that's invisible when you read the train, but it tells the computer exactly where the train ends.

The Formula (the four you'll use most)

FunctionWhat it does
strlen(str)counts characters, not including \0
strcpy(dest, src)copies src into dest
strcat(dest, src)appends src onto the end of dest
strcmp(s1, s2)0 if equal, negative if s1<s2, positive if s1>s2

Like you're 5

strlen = count the letter-cars (not the secret stop car).

strcpy = copy one train exactly onto a second track.

strcat = hook a second train onto the back of the first one.

strcmp = who comes first in the dictionary — like sorting spelling words for a spelling bee.

Full Worked Example (compiled & run)

A program that runs all four functions on two words and shows exactly what happens.

Like you're 5

We start with two little trains, "Hello" and "World," and watch what each function does to them, one at a time.

Step 1 · The code
#include <stdio.h>
#include <string.h>

int main() {
    char str1[50] = "Hello";
    char str2[50] = "World";
    char str3[50];

    printf("strlen(str1) = %lu\n", strlen(str1));

    strcpy(str3, str1);
    printf("After strcpy: str3 = %s\n", str3);

    strcat(str3, str2);
    printf("After strcat: str3 = %s\n", str3);

    printf("strcmp(str1, str2) = %d\n", strcmp(str1, str2));
    return 0;
}
Step 2 · Trace it through
"Hello" has 5 letter-cars → strlen = 5.
strcpy(str3, str1) makes str3 an exact copy of "Hello".
strcat(str3, str2) hooks "World" onto the back → str3 becomes "HelloWorld".
strcmp compares letter by letter: 'H' (72) comes before 'W' (87), so the result is negative.
Step 3 · Actual program output
strlen(str1) = 5 After strcpy: str3 = Hello After strcat: str3 = HelloWorld strcmp(str1, str2) = -120
The exact negative number from strcmp can vary by compiler (it's only guaranteed to be negative, not a specific value) — but this run genuinely produced −120.

User-Defined Functions (C)

Write a job once, give it a name, and call it by name whenever you need it done again.

The Big Idea

As it actually is (exam version)

A function is a named, self-contained block of code that performs one task. Instead of rewriting the same logic everywhere, you write it once and call it by name whenever you need it.

Structure: return_type function_name(parameter_list) { ...body...; return value; }. It must be declared (a prototype) or defined before it's used.

Like you're 5

Imagine training a helper robot to do one job — say, "make a sandwich." You only teach it once. Any time you're hungry, you just say "robot, make a sandwich" instead of making it from scratch yourself every single time.

The Formula (syntax + how arguments are passed)

return_type function_name(parameter_list) {
    // body
    return value;   // skip this line if return_type is void
}

Call by value: the function gets a copy of your variable — changes inside the function don't affect the original.

Call by reference (using pointers, from the Pointers tab): the function gets the variable's address — changes inside the function do affect the original.

Like you're 5

Call by value hands the robot a photocopy of your homework — it can scribble all over the copy, your original stays clean.

Call by reference hands the robot your actual original notebook — now it can really change what's written in it.

Full Worked Example (compiled & run)

A user-defined function that calculates a factorial by calling itself (recursion).

Like you're 5

Factorial of 5 means "5 × 4 × 3 × 2 × 1." Instead of writing that whole multiplication out, the function just asks a smaller version of itself: "what's 4! ?" — and that version asks an even smaller one, all the way down to 1, then the answers multiply their way back up.

Step 1 · The code
#include <stdio.h>

long factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

int main() {
    int num = 5;
    printf("Factorial of %d = %ld\n", num, factorial(num));
    return 0;
}
Step 2 · Trace the recursion
factorial(5) = 5 × factorial(4)
factorial(4) = 4 × factorial(3)
factorial(3) = 3 × factorial(2)
factorial(2) = 2 × factorial(1)
factorial(1) = 1  (this is the stopping point)
Multiplying back up: 1 → 2×1=2 → 3×2=6 → 4×6=24 → 5×24=120
Step 3 · Actual program output
Factorial of 5 = 120
Compiled and run — confirmed correct.
